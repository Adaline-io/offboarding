# Build Folder

This folder holds **brand assets** (Adaline wordmark and signs) used by every per-client offboarding HTML.

Unlike a typical build folder, **there is no build script in this repo**. The offboarding HTMLs are generated by Claude in the chat interface using a reusable Python template (`build_offboarding.py`) that lives upstream. The pattern keeps the repo clean and ensures every new client offboarding uses the latest version of the template.

## To add a new client offboarding

1. **In Claude chat**, send: "Build the offboarding for {client name}, project {project name}, total paid {amount}, delivered {date}, deliverables {list}, accent colour {hex}."
2. Claude builds the HTML and returns a download link.
3. **In this repo**, save the file as `../{client-slug}/index.html` and commit.

## Why the build script is not in this repo

- The template is actively maintained — every refactor (new fields, layout improvements, brand updates) propagates immediately because Claude regenerates with the latest version
- Keeps the repo small and focused on deployment artifacts (the HTMLs and assets) instead of build tooling
- One source of truth for the offboarding template across all future clients

If you want the build script for offline use, ask Claude in chat — Claude can export the current version of `build_offboarding.py` and you can drop it into this folder.
